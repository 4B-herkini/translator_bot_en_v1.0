# TransLit — AI 비즈니스 커뮤니케이션 서비스 정의서 v2.0

> "아무렇게나 입력하세요. 나머지는 저희가 합니다."

---

## 1. 문제 정의

### 현재 시장의 Pain Point

| 기존 도구 | 문제 |
|-----------|------|
| 구글 번역 / 파파고 | 직역 수준. 톤 선택 불가. 비즈니스 맥락 무시. 입력이 지저분하면 출력도 망가짐 |
| ChatGPT / Gemini 직접 사용 | 매번 프롬프트 작성 필요. 톤·포맷 지시를 반복해야 함. 출력 품질 들쭉날쭉 |
| 크롬 AI 모드 | 범용 AI. 번역 특화 워크플로우 없음. 매번 수동 지시 필요 |
| Gemini Gems / GPTs | 프롬프트 컨테이너일 뿐, 실제 서비스 아님. 음성 입력·정제·다국어 동시 출력 불가 |

### 핵심 문제 한 줄 요약
> 한국어는 교착어라 어순이 자유롭고 구어체가 파괴적인데, 기존 번역 도구는 **정제된 입력을 전제**로 설계되어 있다.

---

## 2. 서비스 포지셔닝

### "번역기"가 아니라 "비즈니스 커뮤니케이션 엔진"

기존 정의서에서 가장 큰 약점은 **"이건 제품이 아니라 기능이다"**라는 지적이었다.
번역은 기능이고, 이 서비스가 파는 것은 **"비즈니스 소통을 즉시 완성해주는 경험"**이다.

| 번역기 | TransLit |
|--------|----------|
| 문장을 번역해줌 | **보낼 수 있는 이메일/메시지를 완성해줌** |
| 입력 품질에 의존 | 입력 품질 무관 |
| 1개 결과 | 상황별 복수 결과 + 선택 |
| 매번 새로 시작 | 히스토리·용어집·맥락 축적 |

## 3. 솔루션

### 핵심 플로우

```
사용자 입력 (음성 or 텍스트, 문법·맞춤법·어순 무관)
    │
    ▼
[1단계] 원문 정제 — 맞춤법, 어순, 띄어쓰기, 문장 구조 교정
    │           └→ 정제된 원문 표시 (사용자 검증·수정 가능)
    │                                ↑ 스트리밍 UI: 정제 결과부터 즉시 표시
    ▼
[2단계] 다중 톤 번역 — 동일 문장을 복수의 톤으로 동시 번역
    │
    ▼
[출력] Professional / Friendly / Concise 3가지 버전
       + 원클릭 복사 + 히스토리 자동 저장
```

### 킬 포인트
- **1단계(원문 정제)가 존재하는 번역 서비스는 현재 시장에 없다**
- 사용자는 입력 품질을 신경 쓸 필요 없음 → 음성 입력의 실용성이 극대화됨
- "내가 이렇게 말했는데 이렇게 정리됐구나" → 번역 결과에 대한 **신뢰도 상승**
- 스트리밍 UI로 정제 결과부터 실시간 표시 → **체감 대기 시간 최소화**

## 4. 타겟 사용자

### Primary — 매일 영어 소통이 필요한 직장인/사업자

| 페르소나 | 상황 | 니즈 |
|----------|------|------|
| 해외 클라이언트 대응 직장인 | 영어 이메일 하나에 30분 소요 | 빠르고 격식 갖춘 번역 |
| 해외 거래처 소규모 사업자 | 견적, 납기, 클레임 등 민감한 소통 | 톤 선택 + 프로페셔널한 출력 |
| 외국인 동료와 일하는 개발자 | 슬랙/지라 영어 코멘트 | 빠른 캐주얼 번역 |
| 한국 기업과 일하는 외국인 | 한국어 존댓말 체계 어려움 | 영→한 역방향 + 존댓말 레벨 지원 |

### Secondary — 확장 가능 시장
- 유학생 (에세이, 교수 이메일)
- 해외직구 셀러 (상품 설명, CS 대응)
- 프리랜서 (해외 플랫폼 클라이언트 소통)

### 시장 규모 추정

| 구분 | 수치 | 근거 |
|------|------|------|
| TAM (전체 시장) | 약 800만 명 | 한국 내 영어 업무 소통 경험 직장인 (사람인 설문 기준 직장인 72%가 영어 업무 경험) |
| SAM (도달 가능 시장) | 약 150만 명 | 이 중 주 3회 이상 영어 이메일/메시지 작성하는 사용자 |
| SOM (초기 목표) | 5,000명 | 출시 12개월 내 유료 전환 목표 |
| 연매출 목표 | 약 6억 원 | SOM 5,000명 × Pro 월 9,900원 × 12개월 |

## 5. 핵심 기능

### MVP (v1.0)

| 기능 | 설명 |
|------|------|
| 양방향 자동 감지 | 한→영, 영→한 자동 판별. 스위치 불필요 |
| 원문 정제 출력 | 입력 원문을 정제된 문장으로 먼저 표시. 사용자 수정 가능 |
| 3톤 동시 번역 | Professional / Friendly / Concise 동시 출력 |
| 음성 입력 | Web Speech API 기반 마이크 입력 |
| 원클릭 복사 | 각 번역 결과 개별 복사 버튼 |
| 스트리밍 UI | 정제 결과부터 실시간 표시 → 체감 대기 시간 최소화 |
| 번역 히스토리 | 최근 50건 자동 저장·검색·재사용 (localStorage 기반) |

| 기능 | 설명 |
|------|------|
| 크롬 익스텐션 | Gmail, 슬랙 등에서 텍스트 드래그 → 바로 번역. 별도 탭 이동 불필요 |
| 단축키 위젯 | 데스크톱에서 전역 단축키로 즉시 호출 |
| 상황별 템플릿 | 견적 회신, 납기 협의, 클레임 대응 등 비즈니스 시나리오별 템플릿 |

### v2.0 — 락인 + 확장

| 기능 | 설명 |
|------|------|
| 다국어 확장 | 영어 외 일본어, 중국어, 스페인어 등 |
| 도메인 모드 | IT / 법률 / 의료 / 무역 등 전문 용어집 적용 |
| 톤 커스터마이징 | 사용자 정의 톤 추가 (예: "우리 회사 스타일") |
| 팀 공유 | 팀 내 번역 히스토리·용어집 공유 |
| 자동 문장 생성 | 상황 키워드만 입력 → 완성된 비즈니스 문장 생성 |

## 6. 서비스 형태 전략

### 단독 웹앱이 아닌, 사용자 동선 안으로 들어가는 구조

3개 AI가 공통으로 지적한 핵심: **"별도 웹사이트를 열어야 하는 구조는 리텐션이 약하다."**

```
[Phase 1] 웹앱 — MVP 검증 + 포트폴리오 시연용
    │
    ▼
[Phase 2] 크롬 익스텐션 — 실 사용자 리텐션 확보 (핵심 전환점)
    │         └→ Gmail, Outlook, Slack 등에서 바로 번역
    ▼
[Phase 3] 데스크톱 앱 (Electron/Tauri) + 모바일 (Capacitor/PWA)
```

크롬 익스텐션이 **Phase 2에서 반드시 나와야 하는 이유**:
- 사용자가 이메일 쓰다가 탭을 옮기는 것 자체가 이탈 포인트
- 드래그 → 번역 → 붙여넣기, 이 동선이 3초 안에 끝나야 함
- 이게 되면 "매일 쓰는 도구"가 됨

## 7. 기술 구조

### 프론트엔드
- HTML / CSS / JavaScript (바닐라 or 경량 프레임워크)
- Web Speech API (음성 인식)
- 스트리밍 응답 UI (SSE / ReadableStream)
- 반응형 디자인 (모바일/데스크톱)

### 백엔드 — 멀티 LLM 라우팅

```
요청 수신
    │
    ├── Free 플랜 → 경량 모델 (Claude Haiku / GPT-4o mini)
    │                └→ 1톤 번역만 제공
    │
    └── Pro/Business → 고성능 모델 (Claude Sonnet / GPT-4o)
                        └→ 3톤 동시 + 음성 + 히스토리
```

| LLM | 역할 | 비고 |
|-----|------|------|
| Claude Sonnet | 기본 엔진 | 한국어 정제 + 번역 품질 우수 |
| GPT-4o | 폴백 #1 | Claude 장애 시 자동 전환 |
| Gemini 2.0 | 폴백 #2 | 이중 폴백 |
| Claude Haiku / GPT-4o mini | Free 플랜 전용 | 원가 절감용 경량 모델 |

시스템 프롬프트는 **3개 LLM 모두에서 동일 품질 출력**되도록 각각 최적화 설계.

### 배포 확장성

```
웹앱 (원본)
    ├── 그대로 → 웹 서비스 (URL 배포)
    ├── 크롬 익스텐션 → 워크플로우 침투 (Phase 2 핵심)
    ├── Electron / Tauri → 데스크톱 앱 (Windows/Mac)
    └── Capacitor / PWA → 모바일 앱 (Android/iOS)
```

### 비용 구조

| 항목 | Free | Pro |
|------|------|-----|
| 모델 | Haiku / GPT-4o mini | Sonnet / GPT-4o |
| 1회 번역 원가 | 약 5~15원 | 약 50~150원 |
| 일 사용 제한 | 3회 | 무제한 (일 평균 30회 가정) |
| 월 원가 | 약 450원 | 약 3,000~4,500원 |

Free 플랜 핵심: **바이럴이 터져도 원가가 폭발하지 않는 구조**. 경량 모델 + 횟수 제한 + 기능 제한으로 3중 방어.

## 9. 경쟁 해자 (Moat) 전략

### 현재 약점: "프롬프트 자동화 레이어"에 불과 → 복제 용이

### 해자 구축 로드맵

```
[Phase 1] 워크플로우 차별화 (복제는 가능하나, 실행력으로 선점)
    │
    ▼
[Phase 2] 데이터 락인 (복제해도 데이터는 못 가져감)
    │   ├── 사용자별 번역 히스토리 축적
    │   ├── 회사별/산업별 용어집 학습
    │   └── 반복 패턴 기반 추천 ("지난번에 이렇게 보내셨는데, 이번에도?")
    │
    ▼
[Phase 3] 도메인 특화 (범용 AI가 따라올 수 없는 영역)
    │   ├── 무역: HS코드, 인코텀즈, LC 용어
    │   ├── IT: 기술 용어, API 문서, 릴리즈 노트
    │   └── 법률: 계약서, NDA, MOU 전문 표현
    │
    ▼
[Phase 4] 네트워크 효과 (팀 기능)
        ├── 팀 용어집 공유 → 한 명이 쓰면 팀 전체가 잠김
        └── 회사 톤 가이드 설정 → 조직 수준의 전환 비용 형성
```

## 10. 보안 및 프라이버시

비즈니스 이메일·계약서·클레임 내용이 입력되는 서비스 특성상 **보안은 선택이 아닌 필수**.

| 항목 | 정책 |
|------|------|
| 데이터 저장 | Free: 서버 미저장 (클라이언트 로컬만). Pro: 암호화 저장 (AES-256) |
| 전송 보안 | HTTPS (TLS 1.3) 필수 |
| AI 학습 제외 | 사용자 입력 데이터는 LLM 학습에 사용되지 않음 명시 (API 사용 시 기본 적용) |
| 데이터 삭제 | 사용자 요청 시 즉시 삭제. 계정 탈퇴 시 30일 내 완전 삭제 |
| Business 플랜 | 전용 테넌트 옵션 제공 가능 |
| 개인정보 처리 | 개인정보 처리방침 수립. 한국 개인정보보호법 준수 |

## 14. 요약

TransLit은 **"입력 품질을 사용자에게 요구하지 않는 최초의 비즈니스 커뮤니케이션 서비스"**다.

교착어 한국어의 특성을 역이용하여, 어순·맞춤법·문법이 파괴된 입력도 AI가 정제한 뒤 번역하는 2단계 파이프라인이 핵심이며, 이는 현존하는 어떤 번역 서비스도 제공하지 않는 구조다.

단독 웹앱에 머무르면 "기능"으로 끝나지만, **크롬 익스텐션으로 사용자 업무 동선 안에 침투**하고, **도메인 용어집·히스토리로 데이터 락인**을 형성하면, 대기업이 범용 기능을 추가해도 대체할 수 없는 **전문 도구**가 된다.

> "좋은 문제를 잡았다. 이제 기능이 아닌 제품으로 만든다."
