# TransLit — AI 비즈니스 번역 서비스 정의서

> "아무렇게나 입력하세요. 나머지는 저희가 합니다."

---

## 1. 문제 정의

### 현재 시장의 Pain Point

| 기존 도구 | 문제 |
|-----------|------|
| 구글 번역 / 파파고 | 직역 수준. 톤 선택 불가. 비즈니스 맥락 무시. 입력이 지저분하면 출력도 망가짐 |
| ChatGPT / Gemini 직접 사용 | 매번 프롬프트 작성 필요. 톤·포맷 지시를 반복해야 함. 출력 품질 들쭉날쭉 |
| 크롬 AI 모드 | 범용 AI. 번역 특화 워크플로우 없음. 매번 수동 지시 필요 |
| Gemini Gems / GPTs | 프롬프트 컨테이너일 뿐, 실제 서비스 아님. 음성 입력·정제·다국어 동시 출력 불가 |

### 핵심 문제 한 줄 요약
> 한국어는 교착어라 어순이 자유롭고 구어체가 파괴적인데, 기존 번역 도구는 **정제된 입력을 전제**로 설계되어 있다.

---

## 2. 솔루션

### 핵심 플로우

```
사용자 입력 (음성 or 텍스트, 문법·맞춤법·어순 무관)
    │
    ▼
[1단계] 한국어 정제 — 맞춤법, 어순, 띄어쓰기, 문장 구조 교정
    │              └→ 정제된 한국어 원문 표시 (사용자 검증 가능)
    ▼
[2단계] 다중 톤 번역 — 동일 문장을 복수의 톤으로 동시 번역
    │
    ▼
[출력] Professional / Friendly / Concise 3가지 버전 + 원클릭 복사
```

### 킬 포인트
- **1단계(한국어 정제)가 존재하는 번역 서비스는 현재 시장에 없다**
- 사용자는 입력 품질을 신경 쓸 필요 없음 → 음성 입력의 실용성이 극대화됨
- "내가 이렇게 말했는데 이렇게 정리됐구나" → 번역 결과에 대한 **신뢰도 상승**

## 3. 타겟 사용자

### Primary — 매일 영어 소통이 필요한 직장인/사업자

| 페르소나 | 상황 | 니즈 |
|----------|------|------|
| 해외 클라이언트 대응 직장인 | 영어 이메일 하나에 30분 소요 | 빠르고 격식 갖춘 번역 |
| 해외 거래처 소규모 사업자 | 견적, 납기, 클레임 등 민감한 소통 | 톤 선택 + 프로페셔널한 출력 |
| 외국인 동료와 일하는 개발자 | 슬랙/지라 영어 코멘트 | 빠른 캐주얼 번역 |
| 한국 기업과 일하는 외국인 | 한국어 존댓말 체계 어려움 | 영→한 역방향 + 존댓말 레벨 지원 |

### Secondary — 확장 가능 시장
- 유학생 (에세이, 교수 이메일)
- 해외직구 셀러 (상품 설명, CS 대응)
- 프리랜서 (해외 플랫폼 클라이언트 소통)

## 4. 핵심 기능

### MVP (v1.0)

| 기능 | 설명 |
|------|------|
| 양방향 자동 감지 | 한→영, 영→한 자동 판별. 스위치 불필요 |
| 한국어 정제 출력 | 입력 원문을 정제된 한국어로 먼저 표시 |
| 3톤 동시 번역 | Professional / Friendly / Concise 동시 출력 |
| 음성 입력 | Web Speech API 기반 마이크 입력 |
| 원클릭 복사 | 각 번역 결과 개별 복사 버튼 |

### v2.0 확장

| 기능 | 설명 |
|------|------|
| 다국어 확장 | 영어 외 일본어, 중국어, 스페인어 등 |
| 도메인 모드 | IT / 법률 / 의료 / 무역 등 전문 용어집 적용 |
| 번역 히스토리 | 이전 번역 저장·검색·재사용 |
| 톤 커스터마이징 | 사용자 정의 톤 추가 (예: "우리 회사 스타일") |
| 팀 공유 | 팀 내 번역 히스토리·용어집 공유 |

## 5. 기술 구조

### 프론트엔드
- HTML / CSS / JavaScript (바닐라 or 경량 프레임워크)
- Web Speech API (음성 인식)
- 반응형 디자인 (모바일/데스크톱)

### 백엔드
- Claude API (Anthropic) — 한국어 정제 + 번역 처리
- 시스템 프롬프트 사전 설계 — 일관된 출력 품질 보장

### 배포 확장성

```
웹앱 (원본)
    ├── 그대로 → 웹 서비스 (URL 배포)
    ├── Electron / Tauri → 데스크톱 앱 (Windows/Mac)
    └── Capacitor / PWA → 모바일 앱 (Android/iOS)
```

## 7. 경쟁 우위 분석

| 비교 항목 | 구글 번역 | ChatGPT 직접 사용 | 크롬 AI | **TransLit** |
|-----------|-----------|-------------------|---------|-------------|
| 입력 정제 | X | X | X | **O** |
| 다중 톤 동시 출력 | X | 매번 지시 필요 | X | **O** |
| 음성 입력 → 번역 | X | X | X | **O** |
| 설치 없이 사용 | O | O | 크롬 한정 | **O** |
| 비용 | 무료 | 월 $20 | 무료 | Freemium |
| 비즈니스 특화 | X | X | X | **O** |

### 경쟁 해자 (Moat)
- **워크플로우 차별화**: 정제 → 번역 2단계 파이프라인은 범용 AI가 기본 제공하지 않는 구조
- **전환 비용**: 도메인 용어집·히스토리가 쌓이면 타 서비스 이동 어려움
- **니치 시장**: 대기업이 진입하기엔 시장이 작고, 개인 사용자에겐 범용 AI가 불편한 영역

## 10. 요약

TransLit은 **"입력 품질을 사용자에게 요구하지 않는 최초의 번역 서비스"**다.

교착어 한국어의 특성을 역이용하여, 어순·맞춤법·문법이 파괴된 입력도 AI가 정제한 뒤 번역하는 2단계 파이프라인이 핵심이며, 이는 현존하는 어떤 번역 서비스도 제공하지 않는 구조다.

대기업은 수억 명 대상 무료 서비스에 LLM 비용을 붙일 수 없고, 범용 AI는 매번 프롬프트를 요구한다. 그 사이의 빈 공간 — **매일 비즈니스 번역이 필요한 전문가 시장** — 이 TransLit의 포지션이다.
